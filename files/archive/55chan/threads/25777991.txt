 >>25778764 \r\n Eu posso te ajudar explicando a parte técnica OP, mas a parte legal eu não faço a menor ideia de como ficaria. \r\n \r\n Existe uma instituição responsável por estabelecer o seu IP dinamicamente através da internet.Ao se conectar com um provedor de internet é feita um estabelecimento de qual o seu IP através de um DHCP. A ideia do DHCP é que a sua máquina conversa com um servidor qualquer requisita um IP a este servidor para que o seu computador possa ser reconhecido. Todas as máquinas do mundo precisam passar por este procedimento (ou serem configuradas manualmente) para serem reconhecidas como máquinas únicas (ou então problemas de conflito entre máquinas ocorreriam). Este procedimento acontece tanto entre o seu computador e roteador em casa quando faz a função de NAT quanto com o seu computador, provedor e a ICANN. O problema que você está passando é justamente que um IP não aponta única e exclusivamente por um computador, ora o IP é dinâmico justamente por conta do fato de que um IP que está sendo alocado em um momento a um computador pode ser alocado em outro momento para outro computador. \r\n Veja o exemplo simples: \r\n Computador 1 tenta se conectar a rede, este computador requisita um IP dinâmico para se conectar a rede ao provedor, o provedor olha na tabela dele quais os ips disponibilizados pela ICANN podem ser alocados para o cliente e indica para o cliente que pode utilizar o IP xxx.xxx.xxx.xxx. O cliente envia uma DHCPRequest (accept) e finaliza a conexão dizendo que vai utilizar aquele IP. O nome do processo acima é chamado de DHCP que, apesar de parecer complexo, é exatamente o que todos os dispositivos fazem automaticamente para os seus usuários. \r\n \r\n Agora imagine que você se desconectou. Aquele 'espaço' de IP está vago para QUALQUER OUTRO USUÁRIO DAQUELE PROVEDOR QUE REQUISITAR AO PROVEDOR. Lembrando que o IP de maneira solitária e sem o horário é o suficiente para provar a localidade de uma maneira generalizada, mas não do usuário \r\n e dispositivo específico. Logo, para ter a prova concreta de que você é o usuário deste IP, seria necessário ter também o horário em que aquele IP foi utilizado. \r\n \r\n Caso queira fazer uma explicação e defesa mais concreta recomendo ler ICANN, provedores de internet e DHCP.
