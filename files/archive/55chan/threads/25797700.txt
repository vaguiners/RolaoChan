 >>25797876 \r\n \r\n Para ajudar ainda mais, trago mais um apoio: recomendo assistir a seguinte palestra: \r\n \r\n https://youtu.be/3k4CMAaNCuk \r\n \r\n O responsável da palestra, em pouco mais de uma hora, demonstra bem o poder do Love2D. Como você disse que não conhece muito sobre Lua, recomendo que você assista o vídeo e vá dando uma lida na Wiki do Love2D no próprio site dele. \r\n \r\n Sobre a questão de sobre como colocar a ideia no papel e do papel para o código, recomendo que faça Requisitos, Funcionalidades e Casos de USO para não se perder. Esses termos vem da engenharia de Software e são extremamente úteis. \r\n \r\n Funcionalidades são, em geral, o que seu jogo fará de principal. Requisitos são 'funcionalidades dentro das funcionalidades', para específicas ainda mais. \r\n \r\n Exemplo: \r\n \r\n Funcionalidade: \r\n [Sistema de Plantação] \r\n \r\n Requisitos: \r\n [Sistema de Plantação: \r\n { Plantar sementes } \r\n { Regar Sementes } \r\n { Adubar Sementes } \r\n { Colher Sementes }] \r\n \r\n Já Casos de Uso são o passo a passo de como uma funcionalidade será executada. \r\n \r\n Exemplo: \r\n \r\n [Caso de Uso para Sistema de Plantação \r\n { 1- O jogador compra sementes em um NPC fazendeiro.} \r\n { 2- O jogar procura um terreno para o plantio } \r\n { 3- O jogador seleciona a enxada no inventário e prepara o solo } \r\n { 4- O jogador seleciona a semente no inventário e USA em frente ao solo } \r\n { 5- Ele rega para garantir um estágio } \r\n { 6- A cada 6 horas dentro do jogo, um estágio avança e é necessário que o jogador regue a semente para que a planta possa avançar para o próximo estágio } \r\n { 7- O jogador usa adubo para garantir uma colheita com uma maior quantidade da cultura } \r\n { 8- Após 5 estágios, a planta está pronta para ser cultivada. } \r\n { 9- O jogador, ao colher, terá 3 frutos adicionados ao inventário ( 5 caso ele tenha utilizado adubo ) e o tile onde está a planta voltará a ser terreno hábil para o plantio, caso o jogado queira repetir o processo }] \r\n \r\n Fazer esta 'documentação' dá trabalho, mas eu te garanto que você nunca mais vai esquecer do que tinha que fazer, ou passar por uma bloqueio criativo.
