 >>27129449 \r\n Eu só gostaria de interromper por um momento. O que você está se referindo como Linux é, na verdade, GNU / Linux, ou como eu comecei a chamá-lo, GNU e Linux. O Linux não é um sistema operacional em si, mas sim outro componente livre de um sistema GNU em pleno funcionamento, tornado útil pelos corelibs do GNU, utilitários de shell e componentes vitais do sistema, compreendendo um sistema operacional completo, conforme definido pelo POSIX. \r\n Muitos usuários de computador executam uma versão modificada do sistema GNU todos os dias, sem perceber. Através de uma peculiar reviravolta de eventos, a versão do GNU que é amplamente usada atualmente é freqüentemente chamada de “Linux”, e muitos de seus usuários não estão cientes de que é basicamente o sistema GNU, desenvolvido pelo Projeto GNU. Existe realmente um Linux, e essas pessoas estão usando, mas é apenas uma parte do sistema que eles usam. \r\n Linux é o kernel: o programa no sistema que aloca os recursos da máquina aos outros programas que você executa. O kernel é uma parte essencial de um sistema operacional, mas inútil por si mesmo; Ele só pode funcionar no contexto de um sistema operacional completo. O Linux é normalmente usado em combinação com o sistema operacional GNU: todo o sistema é basicamente GNU com Linux adicionado, ou GNU / Linux. Todas as chamadas distribuições “Linux” são na verdade distribuições do GNU / Linux.
