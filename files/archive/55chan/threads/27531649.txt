 >>27531697 \r\n Eu gostaria de intervir por um momento. O que você está se referindo como Linux é, na verdade, GNU/Linux, ou como eu tenho chamado ultimamente, GNU mais Linux. Linux não é um sistema operacional por si só, mas um componente livre do sistema completamente funcional GNU, transformado em algo útil pelas corelibs, shell, e outros componentes vitais GNU, formando um sistema operacional completo conforme definido pelo POSIX. \r\n \r\n Muitos usuários usam uma versão modificada do sistema GNU todos os dias sem perceberem. Por um acontecimento peculiar, a versão do sistema GNU amplamente utilizada hoje é chamada de Linux, e muitos de seus usuários não estão cientes de que ele é basicamente um sistema GNU, desenvolvido pelo projeto GNU. \r\n \r\n O Linux realmente existe, e essas pessoas estão utilizando-o, mas ele é apenas parte do sistema que eles usam. Linux é o kernel: o programa no sistema que aloca os recursos da máquina para os programas em execução. O kernel é uma parte essencial do sistema operacional, mas inútil por si só. Ele só pode funcionar no contexto de um sistema operacional completo. O Linux normalmente é usado em combinação com o sistema operacional GNU: o sistema completo é basicamente GNU com Linux adicionado, ou GNU/Linux. Todas as assim chamadas distribuições Linux são na verdade distribuições de GNU/Linux!
